# Agents Guide: NixOS Configuration

This repository contains NixOS configurations for a home lab environment, using Home Manager and K3s.

## Essential Commands

- **Update System**: `sudo nixos-rebuild switch` (Aliased to `nixupdate` in Zsh)
- **Edit Config**: `nixedit` (Aliased to `$EDITOR ~/git/nixos-config`)
- **Cleanup**: `sudo go run ~/git/nixos-config/hack/laundry/main.go` (Prunes Docker/Podman images and Nix garbage)

## Project Structure

- `/hosts`: Machine-specific configurations (named after TMNT characters: `donatello`, `leonardo`, etc.).
- `/common`: Shared modules for system settings, Home Manager, K3s, and networking.
- `/users`: User-specific NixOS settings (e.g., `ddd.nix`).
- `/styles`: CSS for desktop components (e.g., `wofi.css`).
- `/hack`: Miscellaneous scripts and tools.

## Architecture & Patterns

### Configuration Strategy
The project uses a mix of standard NixOS modules and custom options defined in `/common`. 
- **Imports**: Hosts typically import `../common`, `../users/ddd.nix`, and their specific `hardware-configuration.nix`.
- **Custom Options**: Several modules in `common/` define custom options for easy toggling in host files:
  - `bloat.enable`: Toggles Desktop Environment (Sway, Waybar, Firefox, etc.).
  - `sshBox.enable` / `sshBox.doSecurity`: Manages SSH and Fail2Ban.
  - `isK3sNode.enable` / `isK3sNode.isServer`: Configures K3s role and cluster connectivity.
  - `tailscale.enable`: Toggles Tailscale service.

### Home Manager
Home Manager is integrated as a NixOS module in `common/home-manager.nix`. It handles:
- User packages and dotfiles.
- Desktop environment configuration (Sway, Waybar, Wofi).
- Shell aliases and Zsh configuration.
- Note: It fetches its own tarball (pinned to `release-25.05`) rather than using a flake input or system NIX_PATH.

### K3s Cluster
K3s configuration is centralized in `common/k3s.nix`.
- **Secret Management**: Tokens are read from `/home/ddd/.kube/cluster-secret` (ensure this exists on the host).
- **Networking**: Automatically opens necessary firewall ports (6443, 2379, 2380, 8472).
- **Container Runtime**: Configured to use Podman with Docker compatibility.

## Conventions & Gotchas

- **State Version**: `system.stateVersion` varies by host (e.g., `25.05` or `25.11`). Do not change this unless upgrading. **Hardware Config**: Host files in `/hosts` often reference `/etc/nixos/hardware-configuration.nix` directly. This file is generated by NixOS and is not tracked in this repo.
- **External Dependencies**: Zsh configuration (`initContent`) and Tmux plugins depend on a separate `~/git/dotfiles` repository being present.
- **Experimental Features**: `nix-command` and `flakes` are enabled globally, but the repo itself does not currently use a `flake.nix` entry point.
- **Hardcoded Paths**: Several modules (e.g., `k3s.nix`, `home-manager.nix`) use hardcoded absolute paths to `/home/ddd/`.
